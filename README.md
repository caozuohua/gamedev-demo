## 二、游戏核心概念
玩家扮演一名 Python 程序员，通过 **编码（主操作）** 提升经验、金币、技能，逐步解锁新能力和装备，最终成为 Python 大师。核心循环：消耗“精力” → 获得经验、金币、随机掉落 → 升级 → 解锁新内容。

### 2.1 角色属性
- 等级（Lv，1-100）

  - 调试能力（影响掉落概率）
  - 学习速度（影响技能点获取）
  - 创造力（影响特殊事件触发概率）
- 技能点（未分配的技能点数，升级时获得 3 点）

### 2.2 游戏资源
- 装备栏（机械键盘、人体工学椅、双显示器、降噪耳机、咖啡杯）——每件装备提供不同属性加成，可消耗金币购买/升级。
- 技能树（三个分支）：算法、Web开发、数据分析，每个分支 5 级，点满后解锁被动效果。
- 任务系统：每日任务（如“修复 3 个 Bug”、“完成 10 次编码”），完成后获得金币、精力瓶等奖励。

### 2.3 核心操作
- **主按钮“写代码”**：消耗 1 点精力，获得基础经验 = 10 + 代码质量加成，获得金币 = 5 + 随机掉落。概率触发特殊事件（“发现优雅解法” → 双倍经验；“遇到难缠 Bug” → 获得调试点数等）。
- **学习/升级**：在技能界面分配技能点提升技能等级。
- **商店/背包**：购买装备，查看当前加成。

## 五、核心逻辑实现要求
- **精力恢复**：在 `onShow` 和点击“写代码”时检查时间差，每 3 分钟恢复 1 点精力，不超过上限。
- **升级公式**：所需经验 = 100 * (等级 ^ 1.2)，升级后精力上限增加 5，所有属性提升，技能点 +3。
- **经验/金币计算**：基础值 * (1 + 代码质量/100) * 随机系数(0.9~1.1)。代码质量由技能树和装备提供。
- **掉落系统**：每次编码有 20% 概率掉落精力瓶（恢复 5 精力）、金币袋（+20 金币）、技能书（+1 技能点）之一。
- **每日任务重置**：对比存储日期和当前日期，不同则重置任务进度和列表（重新生成任务目标）。
- **成就检测**：每次属性变化后调用成就检测函数，满足条件则解锁并弹窗通知。
